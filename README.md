# PASS (Particle Accelerator Simulation Studio)



# Introduction

Particle accelerator beam dynamics simulation platform



# How to Use

## Environment Configuration

1. Install Visual Studio (Windows) / GCC (Linux), minimum C++ version 17.

2. Install CUDA (version 12.2 or higher recommended).

3. Install CUDSS ([cuDSS 0.7.1 Downloads | NVIDIA Developer](https://developer.nvidia.com/cudss-downloads)). After installation, add it to your system environment variables, for example:

- Windows: C:\Program Files\NVIDIA cuDSS\v0.7\bin\13

Also, modify the cuDSS directory in CMakeLists.txt according to the existing paths.

4. Install Python (version 3.12 or higher recommended) and the tfs-pandas library.
5. Install CMake (version 3.29 or higher recommended).
6. Modify the file paths in the CMakeLists.txt file according to your software installation path.

## Compilation Process

1. Navigate to the PASS folder and create a new folder named "build".
2. Navigate to the "build" folder and execute the following commands to build the project:

​	Particle classes containing phase information (high memory usage, suitable for incoherent tune analysis): `cmake ..`

​	Particle classes not containing phase information (low memory usage, not suitable for incoherent tune analysis): `cmake -DPASS_CAL_PHASE=OFF ..`

3. Execute the following command to compile the project: `cmake --build . --config Release -j16`

## Input file

1. Navigate to the `para` folder, modify the parameter configuration in the `generate_main.py` file, run the program, and generate the input file, such as `beam.json`.

2. Optical files can be entered manually, or you can input Twiss files, Error files, etc., generated by MADX in TFS format. If reading MADX files, it is recommended to include the following columns in the MADX file and set the optical calculations to be performed in the component center:

   ```
   FLAG=TWISS,COLUMN=NAME,KEYWORD,S,L,BETX,ALFX,MUX,BETY,ALFY,MUY,X,Y,DX,DPX,K0L,K1L,K2L,K3L,E1,E2,H1,H2,HGAP,FINT,FINTX;
   TWISS, CENTRE, FILE=twiss.tfs;
   ```

## Software Operation

1. In Windows, the program is located in the build/Release folder. Execute the command: .\build\Release\PASS.exe --beam0=..\para\beam.json
2. In Linux, the program is located in the build folder. Execute the command: ./build/PASS --beam0=../para/beam.json
